fennec — AI usage observability for your org

fennec captures AI usage events on this machine and sends them to your org's fennec instance over HTTPS so engineering leaders can answer: who is prompting how, where the money goes by project, and is the right model being used for each task?

What gets captured

  • Claude Code prompts and responses (6 hook events)
  • Anthropic token counts (input, output, cache creation, cache read)
  • Workspace metadata (current directory, git remote, git branch)

What gets redacted before any data leaves this machine

  • Secrets matching the gitleaks v8.21.0 default ruleset (~181 patterns)
  • Plus 4 fennec-supplemental rules: Anthropic API keys, opaque Bearer tokens, PEM headers, GCP API keys

Prerequisite: Node.js 22 LTS or later on PATH

The fennec daemon wrapper at /usr/local/fennec/bin/fennec exec's node from the system PATH. If Node is not installed, the postinstall script will warn you and the daemon will not start until you install it.

  • Recommended: brew install node@22
  • Or download from https://nodejs.org/

After installation, see the included PRIVACY.md for full details on what's captured, how it's redacted, and where the data flows.
