<objective>
Wire the five adapters built by Plans 02-01/02-02/02-03 into the running daemon. `daemon/src/cli/daemon.ts` step 7 is the ONLY place adapters actually get registered and started — adapter classes do NOT self-register on import — so today the CodexAdapter/GeminiAdapter/CursorAdapter/GitAdapter (in-process file-watchers) and the BridgeEventsAdapter (out-of-process Copilot/browser ingress) are BUILT but dead. This plan adds `registry.register(new <Adapter>(...))` calls for all five alongside the existing `ClaudeCodeAdapter` registration, before `registry.startAll()`, so a live prompt on any surface produces a captured event.

Purpose: The single runtime wiring point that makes CAP-03/04/05/06/09 live capture actually run. It depends on the adapter classes existing (02-01 builds BridgeEventsAdapter + the /v1/events bridge route; 02-02 builds Codex/Gemini; 02-03 builds Cursor/git), so it runs in Wave 3 after those complete. It owns `daemon/src/cli/daemon.ts` exclusively — the building plans deliberately deferred registration here to avoid a file-ownership conflict.

Output: the step-7 registration block in `daemon/src/cli/daemon.ts` (five new `registry.register(...)` calls + imports), and daemon.test.ts assertions that each adapter is registered before startAll and the daemon still boots + shuts down cleanly.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| adapter class → running daemon | An unregistered adapter is dead code; registration is the runtime activation point — a missed registration is a silent capture gap (heartbeat still fires, so failure is visible) |
| bridge "events" subscription → capture | BridgeEventsAdapter must subscribe before bridge.start so the first /v1/events POST has a listener |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-01 | DoS / Repudiation | An adapter is built but never registered → a whole capture surface silently produces zero events | HIGH | mitigate | This plan registers all five in daemon.ts step 7; daemon.test.ts asserts each is registered before startAll (Task 2); heartbeat (CAP-14) makes a genuine zero-capture visible vs a wiring gap |
| T-08-02 | Tampering | BridgeEventsAdapter subscribed AFTER bridge.start → first /v1/events POST dropped | MEDIUM | mitigate | Registration + startAll happen BEFORE bridge.start(port) (step 7 before step 8); the existing comment enforces "listener attached before first POST" (Task 1) |
| T-08-03 | DoS | A new adapter throwing at start() crashes daemon boot | MEDIUM | mitigate | Adapters follow the Adapter contract (start tolerates absent watch targets — zero-capture is valid); daemon.test asserts clean boot with all registered (Task 2); adapter drop-on-throw is the registry's job (S2, owned by 02-02/03) |
