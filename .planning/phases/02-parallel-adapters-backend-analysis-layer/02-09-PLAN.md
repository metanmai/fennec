<objective>
Build the GitHub Copilot capture surface (CAP-06): a `vsce`-packaged VS Code sidecar extension workspace (`vscode-extension/`) that reads Copilot's local chat cache and forwards it to the daemon. Copilot's built-in chat is closed to the public VS Code APIs (Chat Participant only sees `@`-routed prompts), so the viable path is reading `workspaceStorage/<md5>/chatSessions/<uuid>.json` on a timer, diffing by `requestId`, and POSTing new requests to the daemon's loopback `/v1/events` bridge (added in 02-01) authenticated by the pairing token. The daemon's BridgeEventsAdapter (02-01) normalises these into `tool: "copilot"` events. CAP-06 is a HARD requirement with no defer clause — the workspace must exist and be exercised locally.

Purpose: One of the six capture surfaces (CAP-06). It is an out-of-process client — a brand-new workspace with no in-repo analog beyond the POST-to-bridge shape.

Output: the `vscode-extension/` workspace (package.json, tsconfig, extension entry, chatSessions reader + tests, bridge client + tests, resolve-paths, a fixture chatSession, README) — built and `vsce package`-able; the daemon-side normalisation is already in 02-01.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Copilot chatSessions file → sidecar | Untrusted, version-fragile JSON written by VS Code/Copilot; read-only |
| sidecar → daemon /v1/events | The sidecar POSTs to 127.0.0.1 with a pairing token; the bridge authenticates the header |
| pairing token → extension storage | The token is stored in extension config/globalState (browser-inaccessible on-disk secret cannot be read) |
