## Details

### C7 — Codex double-line (Pitfall 1)
A real rollout file shows `grep -c user_message`=1 and `grep -c '"type":"message"'`=3 (3 includes assistant messages). The single user prompt is represented as both a `response_item`+`payload.type:message`(role user) AND an `event_msg`+`payload.type:user_message`. Emit only on the former (synapse's choice) to map one prompt → one row.

### C11 — git revert reflog prefix is `revert:` not `commit (revert):`
Controlled repo (`git -c core.editor=true revert --no-edit HEAD`) produced reflog `HEAD@{1}: revert: Revert "second change"`. The RESEARCH A4.2 text listed `commit (revert):` (an older/alternate form). Both the `revert:` prefix AND the `Revert "<subject>"` commit-subject rule (D2-06) catch it; the git-watcher should match on the `Revert "` subject (robust across git versions) and/or the `revert:`/`commit (revert)` reflog prefixes.

### C16 — btree_gist EXCLUDE proof (local PG 17.10)
```
CREATE EXTENSION btree_gist; -- OK
ALTER TABLE model_pricing ADD CONSTRAINT ... EXCLUDE USING gist (
  model WITH =, token_kind WITH =, tstzrange(effective_from,effective_to,'[)') WITH &&
) WHERE (pricing_kind = 'per_token'); -- OK
-- 2 non-overlapping rows for sonnet-5/input: both INSERT 0 1
-- overlapping row: ERROR conflicting key value violates exclusion constraint model_pricing_no_overlap
-- price@2026-08-15 -> 2.0 ; price@2026-10-15 -> 3.0  (effective-date selection works)
```

### C36 — one push consumer per queue
Local `wrangler deploy --dry-run` with two `consumers` entries on the SAME queue did NOT error at config-validation (it parsed fine). The 1-push-consumer-per-queue limit is a Cloudflare account/deploy-time constraint, not a wrangler config-schema rule, so it cannot be confirmed locally. Treat as a documented platform limit (RESEARCH C7.4). Design implication holds: prefer Option A (one consumer Worker running correlation + model-fit as two idempotent functions) — refines D2-14.

### C40 — Gemini ephemeral
`~/.gemini/tmp/` contains project dirs (`probe-gemini`, `zsh-pro`) but `find ~/.gemini -name '*.jsonl'` returns 0 files right now. The transcript directory structure is confirmed; the live `chats/session-*.jsonl` filename and any token field must be re-observed by running a live `gemini` prompt at build time (Q9). The ephemerality risk (chats age out between sessions) is confirmed real.

### C41 — subscription pricing figures (REFUTED SPEC numbers)
SPEC line ANL-09 / requirement 16: "Subscription products (Copilot $19/mo, ChatGPT Pro $20/mo)". RESEARCH C11.7 corrects: Copilot **Pro $10/mo** (moved to usage-based/credit billing 2026-06-01; Pro+ $39, Max $100); ChatGPT **Plus $20/mo** (Pro is $200, a $100 tier launched 2026-04-09 per research). The SPEC's "$19 Copilot" is stale and "ChatGPT Pro $20" mislabels Plus as Pro. Because pricing lives in the `model_pricing` effective-date table (ANL-08, no hardcoded constants), this is a seed-data correction, NOT a schema change. The exact live figures remain volatile/UNVERIFIABLE here — re-verify at build (Q12). Material for the cost worker's seed migration, not its logic.

### C42 — Copilot modelId not uniform
Across 9 sessions / 99 requests: `modelId` present on 98 (one null), with prefixes `copilot/` and `github.copilot-chat/`. Normaliser must (a) tolerate null `modelId` (skip or tag unknown), and (b) strip either prefix to get the bare model name for the model→tier lookup.
