<objective>
Implement the `fennec inspect` transparency CLI (CAP-18): a new `daemon/src/cli/inspect.ts` subcommand wired into the `daemon/src/index.ts` dispatcher that reads the local JSONL queue and prints every event captured in the last 24 hours — `tool`, `occurred_at`, `kind`, and the redacted `payload` — then prints the destination backend URL. Because the registry redacts synchronously before the queue write, the queue contents are exactly what leaves the machine, so `inspect` is the honest "here is what fennec sends" surface. A canary test proves zero raw secret characters appear.

Also implement a dedicated least-privilege `fennec pair` command (`daemon/src/cli/pair.ts`): it reads the daemon's shim/pairing secret and prints ONLY the token, for one-time copy-paste into the VS Code sidecar (Plan 09) and browser-extension (Plan 10) options. This RECONCILES the Plans 09/10 assumption that a command surfaces the token: `fennec inspect` stays strictly read-only and NEVER prints a secret; `fennec pair` is the single, purpose-built, secret-surfacing command (H-MEDIUM pairing-surface reconcile).

Purpose: The developer-facing transparency guarantee under the tamper-resistant, no-pause posture (D-16 removed CAP-17) plus the extension/sidecar pairing entry point. This is the fifth ROADMAP Phase-2 success criterion. It depends only on Phase-1 daemon queue infra + the loopback-bridge secret-store + Plan 01's stable shared work; it owns files disjoint from Plans 02/03 (only `daemon/src/index.ts`, which no other Phase-2 plan touches).

Output: `daemon/src/cli/inspect.ts` + `daemon/src/cli/pair.ts` + co-located tests (incl. the CAP-18 canary), and the dispatcher cases + printUsage lines + public-API exports in `daemon/src/index.ts`.
</objective>
