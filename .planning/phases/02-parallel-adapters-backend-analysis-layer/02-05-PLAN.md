<objective>
Implement the `fennec inspect` transparency CLI (CAP-18): a new `daemon/src/cli/inspect.ts` subcommand wired into the `daemon/src/index.ts` dispatcher that reads the local JSONL queue and prints every event captured in the last 24 hours — `tool`, `occurred_at`, `kind`, and the redacted `payload` — then prints the destination backend URL. Because the registry redacts synchronously before the queue write, the queue contents are exactly what leaves the machine, so `inspect` is the honest "here is what fennec sends" surface. A canary test proves zero raw secret characters appear.

Purpose: The developer-facing transparency guarantee under the tamper-resistant, no-pause posture (D-16 removed CAP-17). This is the fifth ROADMAP Phase-2 success criterion. It depends only on Phase-1 daemon queue infra + Plan 01's stable shared work; it owns files disjoint from Plans 02/03 (only `daemon/src/index.ts`, which no other Phase-2 plan touches).

Output: `daemon/src/cli/inspect.ts` + co-located `inspect.test.ts` (incl. the CAP-18 canary), and the dispatcher case + printUsage line + public-API export in `daemon/src/index.ts`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local JSONL queue → inspect output | inspect reads the on-disk queue (already redacted) and renders it to the developer's terminal / --json |
| developer → capture transparency | inspect is the honest disclosure surface — it must show exactly what leaves the machine, nothing more, nothing hidden |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-01 | Information disclosure | inspect surfaces a RAW secret from a captured prompt | HIGH | mitigate | inspect reads the ALREADY-REDACTED queue (registry redacts before appendEvent — EVIDENCE C25); no raw-secret path exists; canary test asserts zero raw canary chars in output (Task 1, CAP-18 acceptance) |
| T-05-02 | Spoofing / Info disclosure | inspect makes a network call that could exfiltrate or show unredacted data | MEDIUM | mitigate | inspect is local-only — reads replayFromWatermark, never calls the backend (D2-29); grep asserts no fetch/http in inspect.ts (add to test if desired) |
| T-05-03 | Tampering | inspect mutates or clears the queue (a hidden pause) | LOW | mitigate | inspect is strictly read-only; no appendEvent/write path; no pause/disable command (CAP-17 removed, D2-29); grep asserts no `case "pause"` (Task 2) |
| T-05-04 | Repudiation | inspect hides the destination, so the dev can't see where data goes | LOW | mitigate | inspect always prints env.apiBaseUrl (D2-29); grep asserts apiBaseUrl referenced (Task 1) |
| T-05-SC | Tampering | New npm install | LOW | accept | No new dependency — inspect uses existing daemon stdlib + replayFromWatermark + CANARIES; nothing installed |
