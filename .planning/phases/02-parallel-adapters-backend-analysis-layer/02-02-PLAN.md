<objective>
Build the Codex CLI and Gemini CLI transcript-watcher adapters: two chokidar-based, tail-on-append, in-process adapters that implement the locked `Adapter` contract and emit `tool: "codex"` / `tool: "gemini"` events through the registry `emit` callback. Each parses the upstream transcript JSONL with per-line version tolerance, maps a user prompt to a single `prompt_submitted` event, and counts malformed lines as `parse_errors` without crashing.

Purpose: Two of the six new capture surfaces (CAP-03, CAP-04). They are file-watchers over local transcript files, fully parallel to the Cursor/git adapters (Plan 03) since they own disjoint files.

Output: `daemon/src/adapters/codex/` and `daemon/src/adapters/gemini/` (adapter + normaliser + resolve-paths + co-located tests) plus checked-in fixture transcripts.
</objective>

| adapter → registry emit | Adapter must never bypass the registry's redaction/queue pipeline |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-09 | DoS | Malformed transcript line crashes the watch loop / spins | MEDIUM | mitigate | normaliser throws → registry counts parse_errors + drops (S2); adapter swallows via void forward().catch; test covers a garbage line (both tasks) |
| T-02-10 | Information disclosure | Secret in a captured prompt reaches the queue unredacted | HIGH | mitigate | Adapter emits via registry only (D2-01); registry redacts BEFORE queue.append (PRIV-01, EVIDENCE C25) — adapter never touches redactor/queue; verified by the existing registry redaction invariant |
| T-02-11 | Tampering | Adapter writes/migrates the upstream transcript | LOW | accept | Adapters open files read-only (chokidar watch + fs read); no write path exists in the adapter code |
| T-02-SC | Tampering | No new deps beyond chokidar (added in Plan 01) | LOW | accept | These adapters import the chokidar already vetted in Plan 01; no additional installs |
</threat_model>
