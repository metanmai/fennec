<objective>
Build the Codex CLI and Gemini CLI transcript-watcher adapters: two chokidar-based, tail-on-append, in-process adapters that implement the locked `Adapter` contract and emit `tool: "codex"` / `tool: "gemini"` events through the registry `emit` callback. Each parses the upstream transcript JSONL with per-line version tolerance, maps a user prompt to a single `prompt_submitted` event, and counts malformed lines as `parse_errors` without crashing.

Purpose: Two of the six new capture surfaces (CAP-03, CAP-04). They are file-watchers over local transcript files, fully parallel to the Cursor/git adapters (Plan 03) since they own disjoint files.

Output: `daemon/src/adapters/codex/` and `daemon/src/adapters/gemini/` (adapter + normaliser + resolve-paths + co-located tests) plus checked-in fixture transcripts.
</objective>
