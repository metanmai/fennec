## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| raw event rows → aggregator | the cron reads ai_events/git_events/prompt_outcomes/model_fit_scores and must scope by org_id from those rows |
| model_pricing rows → cost math | prices are data; a hardcoded price in code would break the effective-date single-source-of-truth (ANL-08) |
| cron trigger → DB | a cron run writes rollups; a re-run must replace, not duplicate, the day |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-01 | Tampering | Hardcoded price constant bypasses model_pricing (ANL-08 anti-pattern) | HIGH | mitigate | no-hardcoded-price grep guard on cost.ts + aggregator.ts (Task 1, S9); prices only from getEffectivePrice reading model_pricing |
| T-07-02 | Tampering | Cache tokens collapsed into one figure (LiteLLM 70%-miscount, ANL-06) | HIGH | mitigate | four distinct token×price terms in computeEstimatedCost; test proves cache_creation and cache_read are independent multiplications (Task 1/D2-24) |
| T-07-03 | Repudiation | Subscription cost silently folded into cost_estimated (ANL-09 anti-pattern) | MEDIUM | mitigate | cost_subscription is a distinct rollup column computed separately; test asserts it is NOT in cost_estimated (Task 2/D2-27) |
| T-07-04 | Tampering | Re-running the cron double-counts a day | MEDIUM | mitigate | idempotent upsert `ON CONFLICT ... DO UPDATE` replaces the day's rows (Task 2/D2-23); test asserts re-run yields the same rows |
| T-07-05 | Information disclosure | Cross-tenant rollup rows | HIGH | mitigate | aggregator groups + writes by org_id from the source rows, never a client-supplied value (T-05-02); parameterized queries (T-05-03) |
| T-07-06 | Tampering | SQL injection in aggregator/rollup/pricing queries | HIGH | mitigate | parameterized `$1..$N` only in model-pricing.ts + daily-rollups.ts + aggregator queries (T-05-03) |
| T-07-07 | DoS | Cron aggregates the whole table every run (unbounded scan) | LOW | mitigate | queries scoped to the target day's partition (occurred_at range); document the day-bounded query in the SUMMARY |
| T-07-SC | Tampering | New npm install | LOW | accept | No new runtime dependency — uses existing `pg`, `@fennec/shared`, Cloudflare cron trigger (no binding install); nothing installed |
