Git (EVIDENCE C11/C12/C21, RESEARCH A4):
- Watch `.git/HEAD` (~21 bytes; branch switch) + `.git/logs/HEAD` (reflog; appended on commit/checkout/revert), tailing the reflog from a PERSISTED byte offset (H13). Working-tree `file_edit` from a DEBOUNCED `git status --porcelain` poll (H-LOW — the lowest-confidence event_type; see below).
- event_type from reflog message: `commit: <subj>`→commit; subject matching `^Revert "`→revert (PRIMARY rule, version-robust) ALSO accept reflog prefix `revert:` (modern git, C11) and `commit (revert):` (older); `checkout: moving from A to B`→branch_switch. `file_edit` is NOT derived from the reflog — it comes from the porcelain poll.
- file_edit (LOWEST confidence, H-LOW): a debounced `git status --porcelain` poll (triggered on a working-tree chokidar change, debounced ~a few seconds so a burst of saves coalesces). When porcelain reports uncommitted changes that were not present at the previous poll, emit ONE `file_edit` per debounce window (do NOT emit one per changed file — coalesce). This is explicitly the least-reliable event_type and is documented as such; it must never crash the watch loop (porcelain failure → soft skip).
- Context via porcelain (C12): `git config --get remote.origin.url`, `git rev-parse --abbrev-ref HEAD`, `git show -s --format=%cI HEAD` (ISO8601 occurred_at for a commit; new Date().toISOString() for branch_switch/file_edit).
- git_events.event_type CHECK = IN (commit, revert, file_edit, branch_switch) (C21). NO schema_version bump (git kinds NOT in EventKindSchema — D2-06/C20).
