<objective>
Build the macOS installer pipeline (signed `.pkg`, Apple notarised, MDM-configurable), the LaunchDaemon + Helper LaunchAgent plists, and the three CLI commands (`fennec wizard`, `fennec init`, `fennec uninstall`) that drive personal-tier + org-tier installs. The signed `.pkg` from this plan is what plan 10 runs against to prove the smoke test end-to-end.

Purpose: Seven Phase 1 requirements close out here — DAE-01 (`fennec wizard` personal interactive), DAE-02 (`fennec init` MDM non-interactive), DAE-05 (LaunchDaemon system-level), DAE-08 (Apple-notarised .pkg), DAE-12 (signed .pkg replaces npm-global), DAE-21 (MDM Configuration Profile primitives), PRIV-07 (first-run consent screen — both wizard interactive AND init logged audit per Pitfall 8). The Helper LaunchAgent registration also satisfies DAE-20's UX path (Helper LaunchAgent is the notifier from plan 08).

Output: A working `make installer` (or equivalent) that produces `installer/build/fennec.pkg` which is signed, notarised, stapled, MDM-configurable, and verified by `spctl --assess`. CLI commands exist as TypeScript modules with `@clack/prompts` for the wizard. Postinstall script orchestrates the install dance. A `docs/PRIVACY.md` provides the URL referenced by the first-run consent screen.

This plan does NOT run the installer end-to-end on a fresh macOS VM — that's a checkpoint:human-verify in plan 10. This plan delivers the pipeline; plan 10 verifies the smoke test against the produced `.pkg`.
</objective>

## CLI binary entry-point (daemon/src/index.ts)

  The daemon already has src/index.ts (placeholder from plan 01). This plan extends it to dispatch CLI subcommands:

    Usage:
      fennec wizard                                 — interactive personal-tier install
      fennec init --install-secret <secret>          — non-interactive MDM-driven install
      fennec init --read-config /path/to/profile.plist — read install_secret from Configuration Profile
      fennec init                                    — read from /Library/Managed Preferences/dev.fennec.daemon.plist (default MDM-deployed path)
      fennec uninstall --org-token <t>               — org-tier uninstall (audited)
      fennec uninstall                               — personal-tier uninstall (requires sudo)
      fennec daemon                                  — run the daemon process (invoked by LaunchDaemon plist; not user-facing)

  The dispatcher inspects argv[2] and routes to the appropriate module. On unknown subcommand: print usage + exit 1.

## fennec wizard (daemon/src/cli/wizard.ts) — DAE-01

  Interactive flow using @clack/prompts:
    1. Intro: "Welcome to fennec — AI usage observability for your org."
    2. Show consent screen (consent.ts.renderInteractive() — see below). Wait for explicit "Yes I consent" / "No" prompt.
       If "No" → exit 1 with "Consent declined. fennec not installed."
    3. Ask: "Do you have an org install secret (from your IT admin)?"
       If yes → text-input install_secret. Validate >=32 chars (Zod schema EnrollRequestSchema).
       If no → "Personal-tier mode" — auto-generate install_secret (32 bytes urandom) AND auto-create a single-member org for the local user (this happens server-side via a special enrollment flag).
    4. Call enrollDaemon (from plan 08) with the secret.
    5. Persist api_key to /var/db/fennec/key via persistApiKey.
    6. Write LaunchDaemon plist (service/launchdaemon.ts).
    7. Write Helper LaunchAgent plist (service/helper-agent.ts).
    8. Write managed-settings hook entries (managed-settings/install.ts from plan 07).
    9. launchctl load /Library/LaunchDaemons/dev.fennec.daemon.plist.
    10. launchctl load /Library/LaunchAgents/dev.fennec.notifier.plist (per-user; needs `launchctl asuser <uid>`).
    11. Trigger attach flow (attach.ts from plan 08) — opens browser to SSO.
    12. Confirm completion: "Daemon running. Sign in completed. You can quit this wizard."

  REQUIRES: must be run as sudo (writing to /Library/LaunchDaemons, /usr/local/fennec/, /var/db/fennec). Wizard checks process.getuid?.() === 0 at start; if not, prints "Please run with sudo" and exits 1.

## fennec init (daemon/src/cli/init.ts) — DAE-02

  Non-interactive flow:
    1. Parse args: --install-secret <secret> OR --read-config <path-to-mobileconfig> OR default (read from /Library/Managed Preferences/).
    2. Render consent.ts.renderLogged() — writes one-page consent record to /var/log/fennec/first-run-consent.txt with timestamp, machine_id, install_secret_org_name (resolved from enroll response — sequencing means consent is logged BEFORE first hook fires, satisfying PRIV-07 per Pitfall 8). ACTUAL sequencing: write a placeholder consent log BEFORE enrollment + the install metadata; then enrich after enrollment with org_name.
    3. enrollDaemon.
    4. persistApiKey.
    5. Write plists + load via launchctl.
    6. Write managed-settings hook entries.
    7. Do NOT trigger attach flow synchronously — events captured pre-attach are tagged unknown@${hostname} per D-15; the user gets a tray notification on first boot (Helper LaunchAgent), browser auto-opens to SSO via attach flow.
    8. Exit 0.

  Same uid-0 check as wizard.

## fennec uninstall (daemon/src/cli/uninstall.ts) — DAE-19

  1. Parse args: --org-token <t> OR none (personal-tier requires sudo as gate).
  2. uid-0 check.
  3. If --org-token provided: validate against the stored install_secret_hash (or in the future a separate org_admin_token); on mismatch exit 1. (Phase 1: accept the install_secret itself as the org-token since Phase 3 will add a separate admin token table.)
  4. Read api_key (via readApiKey) — needed to post the audit event.
  5. Read machine_id + hostname.
  6. Post uninstall audit via emitUninstallAudit (plan 08).
  7. Unload Helper LaunchAgent: launchctl unload /Library/LaunchAgents/dev.fennec.notifier.plist (via asuser if needed).
  8. Unload LaunchDaemon: launchctl unload /Library/LaunchDaemons/dev.fennec.daemon.plist.
  9. Remove managed-settings entries: removeFennecHooks (plan 07).
  10. Delete binaries: /usr/local/fennec/bin/* (shim, notifier), /usr/local/fennec/lib/* (daemon JS bundle).
  11. Delete api_key: /var/db/fennec/key (and the parent dir if empty).
  12. Delete shim-secret: /etc/fennec/shim-secret.
  13. Delete plists: /Library/LaunchDaemons/dev.fennec.daemon.plist, /Library/LaunchAgents/dev.fennec.notifier.plist.
  14. Print "fennec uninstalled. Synapse and other hooks (if any) preserved." (per D-24).
  15. Exit 0.

  Audit BEFORE filesystem teardown so the audit event reaches the backend even if subsequent steps fail.

## Installer pipeline (installer/macos/build-pkg.sh)

  Steps:
    1. cd installer/macos/ and prepare a payload directory at installer/build/payload/ (this is what `pkgbuild` packages):
       - /usr/local/fennec/bin/node          (the system Node 22 binary, or a vendored Node — Phase 1 uses system Node via PATH)
       - /usr/local/fennec/bin/fennec        (symlink to the daemon CLI entry-point)
       - /usr/local/fennec/bin/fennec-hook    (the Go shim binary from plan 07's shim/build/)
       - /usr/local/fennec/bin/fennec-notifier (the Go notifier binary from plan 08's notifier/build/)
       - /usr/local/fennec/lib/daemon/       (the bundled JS — uses `tsc --build` output from daemon/dist/ + node_modules subset)
       - /Library/LaunchDaemons/dev.fennec.daemon.plist
       - /Library/LaunchAgents/dev.fennec.notifier.plist
    2. pkgbuild --identifier dev.fennec --version 0.1.0 --scripts installer/macos/scripts/ --root installer/build/payload --install-location / installer/build/fennec-component.pkg
    3. productbuild --distribution installer/macos/Distribution.xml --resources installer/macos/Resources --package-path installer/build/ installer/build/fennec-unsigned.pkg
    4. bash installer/macos/sign-test-artefact.sh installer/build/fennec-unsigned.pkg "$DEVELOPER_ID_INSTALLER_NAME" (from 01-CERT-STATUS.md)
       — produces installer/build/fennec.pkg signed + notarised + stapled + spctl-asserted (script from plan 03)
    5. Compute + log SHA-256 of installer/build/fennec.pkg

  installer/macos/scripts/preinstall.sh — runs as root before payload installs:
    - Check macOS version (only macOS 13+ supported in Phase 1; else exit 1 with message).
    - Stop existing fennec daemon if installed (launchctl unload + ignore-errors).

  installer/macos/scripts/postinstall.sh — runs as root after payload installs:
    1. Create /etc/fennec/ if missing (mode 0755).
    2. Generate shim-secret: openssl rand -base64 32 > /etc/fennec/shim-secret; chmod 0644 (per Pattern 9 — user-readable so shim can use it).
    3. Create /var/log/fennec/ if missing (mode 0750, chown 0:0).
    4. Create /var/db/fennec/ if missing (mode 0700, chown 0:0).
    5. Read /Library/Managed Preferences/dev.fennec.daemon.plist (macOS auto-installs MDM Configuration Profile here):
       - If exists AND contains `org_install_secret`: invoke `/usr/local/fennec/bin/fennec init` (which reads from default Managed Preferences path).
       - If does NOT exist: leave for user to run `sudo fennec wizard` manually (personal-tier).
    6. Print friendly message about next steps + privacy policy URL.

  Distribution.xml — productbuild input that:
    - References installer/macos/Resources/welcome.html + conclusion.html.
    - Includes pkg-ref pointing at fennec-component.pkg.
    - Sets minimum macOS target (13.0).

  Configuration.plist — macOS Configuration Profile template (matches 01-RESEARCH.md §MDM Configuration Profile Schema lines 1108-1133):
    PayloadType: Configuration
    PayloadContent: array containing the dev.fennec.daemon payload with keys: org_install_secret, org_name, api_url
    Plus: PayloadDisplayName="Fennec Org Configuration", PayloadScope="System"
    NOT signed in Phase 1 — Phase 5 adds the Jamf/Intune-ready signed mobileconfig flavors per D-09.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| signed .pkg → macOS installer | Gatekeeper validates signature + notarisation ticket before executing postinstall |
| postinstall script → /Library, /usr/local, /etc, /var | Postinstall runs as root via Apple's pkg installer mechanism; trusted execution context |
| Managed Preferences (.mobileconfig) → daemon | MDM-deployed profile is delivered by macOS; daemon reads it via `defaults read` |
| sudo gate → fennec uninstall | Process uid==0 enforced before any teardown step |
