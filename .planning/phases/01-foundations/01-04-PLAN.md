Author the Phase 1 Postgres schema and RLS policy migrations against Supabase Postgres 15+. Every customer-data table is tenant-correct (`org_id NOT NULL`) and RLS-enabled from day 1 per D-26 — these cannot be retrofitted without painful data migrations. `ai_events` and `git_events` are range-partitioned by month per ING-05 / ING-06. <!-- These are the authoritative table shapes; this plan writes them as ordered migrations -->

orgs:
  id UUID PRIMARY KEY DEFAULT gen_random_uuid()
  name TEXT NOT NULL
  install_secret_hash TEXT NOT NULL  -- sha256-hex of plaintext secret (server-side hashing)
  install_secret_expires_at TIMESTAMPTZ NOT NULL DEFAULT (NOW() + INTERVAL '90 days')
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()

users:
  id UUID PRIMARY KEY DEFAULT gen_random_uuid()
  email TEXT NOT NULL UNIQUE
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
  -- Phase 3 will add sign-up + password_hash; Phase 1 SQL-seeds one user

org_members:
  org_id UUID NOT NULL REFERENCES orgs(id) ON DELETE CASCADE
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE
  role TEXT NOT NULL CHECK (role IN ('admin', 'member'))
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
  PRIMARY KEY (org_id, user_id)

projects:
  id UUID PRIMARY KEY DEFAULT gen_random_uuid()
  org_id UUID NOT NULL REFERENCES orgs(id) ON DELETE CASCADE
  name TEXT NOT NULL
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()

daemon_machines:
  id UUID PRIMARY KEY DEFAULT gen_random_uuid()
  org_id UUID NOT NULL REFERENCES orgs(id) ON DELETE CASCADE
  machine_id TEXT NOT NULL  -- stable across reboots (IOPlatformUUID on macOS, etc.)
  hostname TEXT NOT NULL
  os TEXT NOT NULL CHECK (os IN ('darwin', 'linux', 'win32'))
  attached_user_id UUID REFERENCES users(id)  -- NULL until SSO attach
  attached_at TIMESTAMPTZ  -- NULL until SSO attach
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
  UNIQUE (org_id, machine_id)

api_keys:
  id UUID PRIMARY KEY DEFAULT gen_random_uuid()
  org_id UUID NOT NULL REFERENCES orgs(id) ON DELETE CASCADE
  daemon_machine_id UUID NOT NULL REFERENCES daemon_machines(id) ON DELETE CASCADE
  token_hash TEXT NOT NULL UNIQUE  -- sha256 of the issued Bearer token; raw never stored
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
  revoked_at TIMESTAMPTZ
  CREATE INDEX idx_api_keys_token_hash ON api_keys (token_hash) WHERE revoked_at IS NULL

ai_events (PARTITIONED):
  idempotency_key TEXT NOT NULL
  org_id UUID NOT NULL
  user_id UUID  -- NULL until SSO attach
  user_id_unknown TEXT  -- "unknown@${hostname}" pre-attach
  tool TEXT NOT NULL
  occurred_at TIMESTAMPTZ NOT NULL
  received_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
  payload JSONB NOT NULL
  schema_version INTEGER NOT NULL
  redaction_applied_at TIMESTAMPTZ NOT NULL
  redaction_version_hash TEXT NOT NULL
  hostname TEXT NOT NULL
  PRIMARY KEY (idempotency_key, occurred_at)  -- partition column must be in PK
  PARTITION BY RANGE (occurred_at)

  Partitions: ai_events_2026_05 ('2026-05-01' .. '2026-06-01'), ai_events_2026_06 ('2026-06-01' .. '2026-07-01')
  Indexes:
    idx_ai_events_org_occurred ON ai_events (org_id, occurred_at)
    idx_ai_events_user_occurred ON ai_events (user_id, occurred_at) WHERE user_id IS NOT NULL

git_events (PARTITIONED, no rows in Phase 1):
  id UUID DEFAULT gen_random_uuid()
  org_id UUID NOT NULL
  user_id UUID
  occurred_at TIMESTAMPTZ NOT NULL
  repo_remote TEXT
  repo_branch TEXT
  event_type TEXT NOT NULL CHECK (event_type IN ('commit','revert','file_edit','branch_switch'))
  payload JSONB NOT NULL
  schema_version INTEGER NOT NULL
  PRIMARY KEY (id, occurred_at)
  PARTITION BY RANGE (occurred_at)
  Initial partition: git_events_2026_05 ('2026-05-01' .. '2026-06-01')

adapter_heartbeats:
  id UUID PRIMARY KEY DEFAULT gen_random_uuid()
  org_id UUID NOT NULL
  daemon_machine_id UUID NOT NULL REFERENCES daemon_machines(id) ON DELETE CASCADE
  adapter TEXT NOT NULL
  adapter_version TEXT NOT NULL
  schema_hash TEXT NOT NULL
  events_parsed INTEGER NOT NULL CHECK (events_parsed >= 0)
  parse_errors INTEGER NOT NULL CHECK (parse_errors >= 0)
  daemon_unreachable_count INTEGER NOT NULL DEFAULT 0
  interval_start TIMESTAMPTZ NOT NULL
  interval_end TIMESTAMPTZ NOT NULL
  received_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
  schema_version INTEGER NOT NULL
  idempotency_key TEXT NOT NULL UNIQUE  -- prevent double-recording of same interval

daemon_audit_events:
  id UUID PRIMARY KEY DEFAULT gen_random_uuid()
  org_id UUID NOT NULL
  daemon_machine_id UUID REFERENCES daemon_machines(id)  -- NULL ok for post-uninstall audits
  hostname TEXT NOT NULL
  reason TEXT NOT NULL CHECK (reason IN ('user_initiated','mdm_revoke','admin_initiated','attach_completed','enrollment_completed'))
  actor TEXT
  occurred_at TIMESTAMPTZ NOT NULL
  received_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
  schema_version INTEGER NOT NULL
  idempotency_key TEXT NOT NULL UNIQUE

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| daemon → backend → Postgres | All event writes pass through backend (api_key bearer auth); never direct daemon → Postgres |
| service_role JWT → Postgres | Backend uses service_role to bypass RLS in Phase 1; middleware in plan 05 enforces org_id |
| seed migration → production | Seed migration is meant for development + smoke test only; the test secret is publicly known and the migration documents this clearly |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | Cross-tenant data leak via missing RLS | mitigate | Every customer-data table has RLS enabled in migration 6 with a placeholder org_id isolation policy. Even though Phase 1's backend uses service_role (RLS bypass), the policies are in place and become load-bearing as soon as Phase 3 introduces user-scoped tokens. Migration 6 fails CI if any table is missing the policy. |
| T-04-02 | Tampering | Idempotency key collision used to overwrite legitimate event | mitigate | `ai_events` PRIMARY KEY is `(idempotency_key, occurred_at)` so duplicate keys with different timestamps don't collide. Plan 05 backend uses `ON CONFLICT (idempotency_key, occurred_at) DO NOTHING`. Collision space is 128-bit hex per CanonicalEvent contract — negligible. |
| T-04-03 | Information Disclosure | Raw API tokens stored in api_keys table | mitigate | `api_keys.token_hash` stores sha256(token); raw token returned ONCE at enrollment time, never re-readable. The seed migration documents the test plaintext only in `.planning/phases/01-foundations/01-SCHEMA-TEST-DATA.md` (kept inside `.planning/` which is fine for dev/test scope; production seeds will use random secrets). |
| T-04-04 | Information Disclosure | Raw install_secret stored in orgs table | mitigate | `orgs.install_secret_hash` stores sha256(plaintext); plaintext is per-org operator-distributed via MDM payload (D-08); rotation policy = "support-only" v1 per Open Question 2. |
| T-04-05 | Denial of Service | Unbounded partition creation overflows planner | mitigate | Phase 1 creates two months of partitions explicitly (current + next). Phase 2+ adds a Supabase scheduled function for dynamic partition creation per ING-05/06 (linked in §Common Pitfalls but out of Phase 1 scope per planner authority). |
| T-04-06 | Information Disclosure | Phase 1 seed test secret committed to git could be mis-deployed to production | mitigate | Seed migration filename includes `seed_phase1_test_data` AND the file contents include a SQL comment block warning "DEV-AND-SMOKE-TEST ONLY. DO NOT APPLY TO PRODUCTION." Production setup docs (Phase 6) instruct operators to skip this migration. |

</threat_model>
