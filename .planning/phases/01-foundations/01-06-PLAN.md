Adapter interface (Pattern 2):
  type Emit = (event: Omit<CanonicalEvent, "redaction_applied_at" | "redaction_version_hash" | "schema_version">) => Promise<void>;
  interface Adapter {
    readonly tool: string;
    readonly version: string;
    start(emit: Emit): Promise<void>;
    stop(): Promise<void>;
  }

AdapterRegistry:
  register(adapter: Adapter): void
  startAll(): Promise<void> — calls each adapter.start(this.makeEmit(adapter))
  stopAll(): Promise<void>
  makeEmit(adapter): Emit — returns a function that:
    1. buildCanonicalEvent({ tool: adapter.tool, adapter_version: adapter.version, ...input }) — fills hostname, os, occurred_at, idempotency_key
    2. redactEvent(event) — synchronous redaction; stamps redaction_applied_at + version_hash
    3. queue.append(redactedEvent)
    4. counter.events_parsed++ (per-adapter)
    On any throw inside this chain: counter.parse_errors++ and the event is DROPPED (not queued). Per Pitfall 1 — better lost event than leaked secret.

LocalQueue (Pattern 4):
  appendEvent(event: CanonicalEvent, queuePath: string) — synchronous fs.openSync(path, "a") + writeSync + closeSync. NEVER async fs.appendFile (no atomic guarantee under crash).
  replayFromWatermark(queuePath, watermark): AsyncIterableIterator<CanonicalEvent> — yields each line after the watermark.
  Watermark file: ~/.fennec/sync-state.json containing { last_synced_event_idempotency_key: string }
  Rotation: when queuePath size > 100MB, rename to events-<ISO>.jsonl and create fresh events.jsonl; sync loop reads from rotated files first.

Redactor (Pattern 3):
  redactEvent(event: CanonicalEvent): CanonicalEvent — synchronous, runs ~150 gitleaks regexes + entropy check, replaces matches with [REDACTED:<RULE_ID>]
  REDACTION_VERSION_HASH = "gitleaks-v8.21-defaults" (constant per shipping ruleset)
  redactEvent stamps redaction_applied_at = new Date().toISOString() and redaction_version_hash = REDACTION_VERSION_HASH
  IF any rule throws: function throws — caller catches and counts parse_errors (DOES NOT swallow per Pitfall 1)

SyncLoop (Pattern 5):
  Constructor: { queuePath, watermarkPath, backendUrl, apiKeyProvider: () => Promise<string>, batchSize: 100, flushInterval: 5000 }
  Internal timer + signal-file watch: events fire OR ~/.fennec/daemon-flush-now exists
  Loop iteration:
    1. Read up to 100 events from watermark forward
    2. POST to ${backendUrl}/api/events/batch with Authorization: Bearer <api_key>
    3. On 2xx: advance watermark to last-event-key; log
    4. On 5xx: exponential backoff (5s, 10s, 20s, ... cap at 60s); do NOT advance watermark
    5. On 4xx: log + advance watermark (the events were rejected as malformed; advancing prevents replay loop)
    6. Honor process.env.NODE_EXTRA_CA_CERTS (Node native via tls.rootCertificates extension) and process.env.HTTPS_PROXY (Node native via undici / hyperdrive proxy support)
    7. On exception during fetch: count toward daemon_unreachable_count for next heartbeat; sleep then retry

Heartbeat (CAP-14, CAP-15):
  Per adapter, every N seconds (default 60):
    Build AdapterHeartbeat { idempotency_key: `${hostname}|${adapter}|${interval_start.toISOString()}`, events_parsed, parse_errors, schema_hash (computed from the most recent payload SAMPLE), daemon_unreachable_count, interval_start, interval_end }
    POST to ${backendUrl}/api/heartbeats with Bearer auth
    Reset events_parsed = 0, parse_errors = 0 for next interval
  Heartbeat emitter STILL runs even when events_parsed has been 0 across N intervals — that's the entire point of CAP-14.

Schema-hash (CAP-15, Open Question 3 — option (a) field-name set hash):
  computeSchemaHash(samplePayload: unknown): string
    1. JSON.parse if string
    2. Recursively collect all keys (Object.keys at every level)
    3. Sort + join with "|"
    4. sha256-hex first 16 chars
  If two consecutive heartbeats report different schema_hash for the same adapter, the backend (Phase 4 dashboard) surfaces "adapter offline / upstream format changed". Phase 1 just ensures the field is in the wire format and is computed correctly.

buildCanonicalEvent(input):
  Stamp hostname (`os.hostname()`), os (`process.platform === "darwin" ? "darwin" : process.platform === "linux" ? "linux" : "win32"`), occurred_at (input.occurred_at ?? new Date().toISOString()), monotonic_seq (per-adapter incrementing counter stored in-memory + persisted to ~/.fennec/seq-<adapter>.json for crash-recovery), idempotency_key (await deriveIdempotencyKey from @fennec/shared)
  schema_version: 1
  redaction_applied_at + redaction_version_hash: filled by redactor downstream

Counter state (per-adapter, in-memory; reset every heartbeat):
  events_parsed: number
  parse_errors: number
  daemon_unreachable_count: number  (incremented per failed sync attempt)
  last_payload_sample: unknown  (used to compute schema_hash at heartbeat time)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| adapter → daemon registry (in-process) | Adapters are part of the daemon process; the boundary is "untrusted payload from upstream tool → daemon pipeline" — redactor sanitises before queue write |
| daemon → JSONL file (filesystem) | Queue file lives under `~/.fennec/` (user-readable but root-owned in production); only daemon writes; corruption window is partial last-line on crash |
| daemon → backend (HTTPS) | TLS terminates at Cloudflare; api_key is the Bearer token; key never logged |
| daemon → corporate proxy (optional) | Node honors NODE_EXTRA_CA_CERTS + HTTPS_PROXY natively per DAE-10 |
