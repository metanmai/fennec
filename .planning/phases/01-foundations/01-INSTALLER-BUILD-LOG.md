| `xcrun notarytool store-credentials fennec-notary` run | TODO | Local dev machine setup |

Once those are filled, the SIGNED build is a one-shot:

```bash
export DEVELOPER_ID_INSTALLER_NAME="Developer ID Installer: <Your Name> (TEAMID1234)"
export APPLE_NOTARY_KEYCHAIN_PROFILE="fennec-notary"
bash installer/macos/build-pkg.sh
```

This will: pkgbuild → productbuild → productsign → `xcrun notarytool submit --wait` → `xcrun stapler staple` → `spctl --assess --type install -vvv`. Expected duration: 5–15 minutes (notarytool wait dominates). On success, `installer/build/fennec.pkg` exists and `spctl --assess` reports `source=Notarized Developer ID`.

Capture the spctl output + final SHA-256 here as a new row.

## On notarisation failure

1. Get the submission ID from the notarytool output.
2. Fetch the log: `xcrun notarytool log <submission-id> --keychain-profile fennec-notary`.
3. Common issues:
   - **Missing entitlements:** the Go binaries (fennec-hook, fennec-notifier) may need a hardened runtime + entitlements plist. The current build does NOT add a hardened runtime to the Go binaries — Phase 1 acceptance is the .pkg signature + notarisation; Phase 5 may add hardened-runtime polish for the embedded binaries.
   - **Symbol-table issues:** `-ldflags="-s -w"` already strips symbols (synapse pattern); if notarytool complains, remove `-s -w` and re-build.
4. Address + re-run.
